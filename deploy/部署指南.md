# 财务RAG问答系统 — 内网服务器部署指南

> 目标：把系统跑在内网云服务器上，同事通过内网网址 `http://服务器IP:8501` 访问，公网访问不到。

---

## 一、首次部署（一次性）

### 1. 准备代码
把本地整个项目拷到服务器，例如放到 `/opt/finance-rag`：
```bash
# 方式A：用 git（推荐，后续升级方便）
git clone <你的内网git地址> /opt/finance-rag

# 方式B：直接拷贝（scp / 共享盘 / U盘）
# 把项目目录上传到 /opt/finance-rag
```

### 2. 建虚拟环境 + 装依赖
```bash
cd /opt/finance-rag
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
```
> ⚠️ 若服务器**完全离线**（连不了 pip 源）：在一台能联网、系统相同的机器上执行
> `pip download -r requirements.txt -d offline_pkgs`，把 `offline_pkgs/` 拷到服务器后
> `.venv/bin/pip install --no-index --find-links offline_pkgs -r requirements.txt`

### 3. 放置嵌入模型（关键，否则启动报错）
系统用 `BAAI/bge-small-zh-v1.5`（约90MB），且强制离线模式。
把本地 `~/.cache/huggingface` 整个拷到服务器对应用户的 `~/.cache/huggingface`：
```bash
# 本地打包
tar czf hf_cache.tar.gz -C ~/.cache huggingface
# 传到服务器后解压到部署用户的家目录
tar xzf hf_cache.tar.gz -C ~/.cache
```

### 4. 配置 .env（密钥与管理员）
```bash
cd /opt/finance-rag
# .env 不在git里，需手动创建
cat > .env <<'EOF'
DEEPSEEK_API_KEY=你的key
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-chat
ADMIN_USER=admin
ADMIN_PASSWORD=改成强密码
# 全站访问口令：部署到服务器/内网必须设置，留空则任何人都能查全部财务数据
APP_ACCESS_PASSWORD=改成访问口令
EOF
chmod 600 .env
```
> 🔴 **确认服务器能访问 `api.deepseek.com`**（公网）。内网若封外网，需让网管放行该域名，
> 否则问答会失败。彻底离线的话需改用内网本地大模型（改 DEEPSEEK_BASE_URL 指向内网地址）。

### 5. 配置开机自启（systemd）
```bash
# 改好 deploy/finance-rag.service 里的 User / 路径
sudo cp deploy/finance-rag.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable finance-rag      # 开机自启
sudo systemctl start finance-rag       # 立即启动
sudo systemctl status finance-rag      # 查看状态
```

### 6. 访问
> ⚠️ `finance-rag.service` 已默认让 Streamlit **只监听 127.0.0.1**（安全，配合下面第 7 步的 nginx+HTTPS）。
> - **推荐**：直接做第 7 步，用 `https://finance.公司内网` 访问。
> - **临时验证/纯内网直连**：把 service 里的 `--server.address` 改回 `0.0.0.0`，再放行端口
>   `sudo firewall-cmd --add-port=8501/tcp --permanent && sudo firewall-cmd --reload`，
>   然后浏览器开 `http://服务器内网IP:8501`（注意此方式为明文传输，不建议长期用于真实数据）。

### 7. Nginx 反代 + HTTPS（🔴 上真实财务数据前必做）
财务数据必须加密传输，不能裸 http。仓库已提供现成模板 `deploy/nginx-finance-rag.conf`（含
Streamlit 必需的 WebSocket 升级头、上传大小、超时、证书获取命令）：
```bash
# 1) 让 Streamlit 只监听本机（deploy/finance-rag.service 已默认 127.0.0.1），对外只走 nginx
# 2) 部署 nginx 配置
sudo cp deploy/nginx-finance-rag.conf /etc/nginx/conf.d/finance-rag.conf
sudo vi /etc/nginx/conf.d/finance-rag.conf     # 改 server_name 和证书路径

# 3) 内网自签证书（公网请用 Let's Encrypt，见模板文末）
sudo mkdir -p /etc/nginx/ssl
sudo openssl req -x509 -nodes -days 3650 -newkey rsa:2048 \
  -keyout /etc/nginx/ssl/finance-rag.key \
  -out    /etc/nginx/ssl/finance-rag.crt \
  -subj "/CN=finance.公司内网"

# 4) 测试并生效
sudo nginx -t && sudo systemctl reload nginx
```
> 之后浏览器用 `https://finance.公司内网` 访问；8501 端口不再需要对外放行（可在防火墙关掉）。
> 并让内网DNS把 `finance.公司内网 → 服务器IP`。

## 二、以后怎么修改完善（日常工作流）

```
①本地笔记本改代码、测试  →  ②提交  →  ③服务器更新+重启
```

### 推荐：用 git
- 本地：改完代码 → `git add . && git commit -m "说明" && git push`
- 服务器：`bash deploy/deploy.sh`（自动备份data、git pull、装依赖、重启）

### 没有内网git：手动覆盖
1. **只拷贝改动的 `.py` 文件**到服务器，覆盖旧文件
2. **绝对不要覆盖** `data/`（知识库数据）和 `.env`（密钥）
3. 重启：`sudo systemctl restart finance-rag`

### 黄金规则
| 能覆盖 | 不能覆盖（覆盖=丢数据） |
|--------|----------------------|
| `*.py`、`requirements.txt`、`.streamlit/`、`deploy/` | `data/`、`.env` |

> `.gitignore` 已把 `data/` 和 `.env` 排除，用 git 时天然不会被动到。

## 三、常用运维命令

```bash
sudo systemctl status finance-rag     # 看运行状态
sudo systemctl restart finance-rag    # 重启（升级后必做）
sudo systemctl stop finance-rag       # 停止
tail -f logs/service.log              # 看实时日志
tail -f logs/service.err              # 看报错日志
```

## 四、排错速查
| 现象 | 原因 | 处理 |
|------|------|------|
| 启动报错找不到模型 | HF缓存没拷或路径不对 | 重做第3步，确认是部署用户的家目录 |
| 问答转圈最后报错 | 连不上 deepseek API | 确认服务器能访问外网/放行域名 |
| 同事打不开网址 | 防火墙没放行端口 | 放行 8501，确认 server.address=0.0.0.0 |
| 升级后数据没了 | 误覆盖了 data/ | 用 backups/ 里的备份恢复 |
| 导出PDF报错 | 没装 fpdf2 或缺中文字体 | `pip install fpdf2`，确认 data/NotoSansSC 字体在 |
